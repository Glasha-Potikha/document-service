# Document Service

Сервис для управления документами с историей статусов и реестром утверждений.

## Технологический стек

- **Java 17** + Spring Boot 3
- **PostgreSQL 15** (в Docker)
- **JPA / Hibernate**
- **Liquibase** (миграции БД)
- **Maven**
- **Testcontainers** (интеграционные тесты)

## Функциональность

- Создание документов со статусом DRAFT
- Просмотр документов (один / списком)
- Перевод статусов: DRAFT → SUBMITTED → APPROVED
- История изменений статусов
- Реестр утверждённых документов
- Поиск с фильтрацией (статус, автор, дата)
- Пакетные операции (до 1000 документов)
- Фоновые workers для автоматической обработки
- Конкурентное утверждение (защита от двойной записи)

## Быстрый старт

### 1. Запуск PostgreSQL через Docker Compose

```bash
# Из корня проекта
docker-compose up -d postgres
```
## Параметры подключения:
- Host: localhost
- Port: 5432
- Database: documentdb
- User: documentuser
- Password: documentpass

### 2. Запуск сервиса
```bash
# Сборка проекта
./mvnw clean package

# Запуск
java -jar target/document-service-1.0.0.jar

# Или через Maven
./mvnw spring-boot:run
```
Сервис будет доступен по адресу: http://localhost:8080 

### 3. Проверка работы
```bash
# Создать документ
curl -X POST http://localhost:8080/api/documents \
  -H "Content-Type: application/json" \
  -d '{"author":"Иван Петров","title":"Важный документ"}'

# Получить все документы
curl http://localhost:8080/api/documents/search
```

### Конфигурация
Основные настройки в application.yml:
```yaml
worker:
  batch-size: 100           # Размер пачки для workers
  submit:
    enabled: true           # Включить SUBMIT-worker
    interval: 30000         # Интервал 30 секунд
  approve:
    enabled: true           # Включить APPROVE-worker
    interval: 45000         # Интервал 45 секунд
```

## API Endpoints
### Документы
- POST /api/documents **Создать документ**
- GET /api/documents/{id}	**Получить документ по ID**
- GET /api/documents?ids=1,2,3	**Пакетное получение**
- GET /api/documents/search	**Поиск с фильтрацией**

### Статусы
- POST /api/documents/submit	**Отправить на согласование**
- POST /api/documents/approve	**Утвердить документы**

### Примеры запросов
#### Поиск с фильтрами:
```text
GET /api/documents/search?status=DRAFT&author=Петров&dateFrom=2026-01-01&page=0&size=20
```
#### Пакетное утверждение:
POST /api/documents/approve
```json
{
  "ids": [1, 2, 3],
  "initiator": "admin",
  "comment": "Утверждено"
}
```
## Фоновые workers
Сервис включает два автоматических workers:
### SUBMIT-worker
- Запускается каждые 30 секунд
- Выбирает пачку DRAFT документов (batch-size)
- Отправляет их на согласование через /api/documents/submit 
- Логирует прогресс и время выполнения
### APPROVE-worker
- Запускается каждые 45 секунд
- Выбирает пачку SUBMITTED документов
- Утверждает их через **/api/documents/approve**
- Логирует результаты (успехи/конфликты/ошибки)

### Логи workers
```text
=== SUBMIT-worker: начало обработки ===
SUBMIT-worker: найдено 100 документов DRAFT
=== SUBMIT-worker: завершение ===
Обработано документов: 100
Успешно отправлено: 100
Время выполнения: 1505 мс
Осталось в очереди: 75
```
## Утилита генерации документов
Отдельный модуль для массового создания тестовых документов.

#### Запуск генератора
```bash
cd generator-utility
./mvnw spring-boot:run -Dspring-boot.run.arguments="--generator.count=1000"
```
### Параметры
- generator.count	Количество документов	10
- generator.batch-size	Размер пачки для логирования	5
- generator.delay-ms	Задержка между запросами (мс)	100
- service.url	URL основного сервиса	http://localhost:8080

### Пример вывода
```text
===========================================
Запуск генерации документов
Всего документов для создания: 1000
===========================================
Создание пачки 1/100 (10 документов)
Прогресс: 10/1000 документов (1%)
...
===========================================
Генерация завершена!
Всего создано: 1000 документов
Общее время: 12500 мс
Средняя скорость: 80 док/сек
```
## Мониторинг по логам
### При создании документов (генератор)
- Всего документов для создания: N - старт
- Прогресс: X/N документов (Y%) - промежуточный прогресс
- Генерация завершена! - финиш с итогами

### При работе workers
- === SUBMIT-worker: начало обработки === - старт пачки
- SUBMIT-worker: найдено X документов - размер пачки
- Обработано документов: X - результат
- Время выполнения: X мс - время пачки
- Осталось в очереди: X - сколько ещё ждут

### Ошибки и конфликты
- Документ X не найден - документ удалён
- Конфликт статусов - неверный статус для операции
- Ошибка регистрации - проблема с реестром

## Тестирование
### Интеграционные тесты
```bash
# Запуск всех тестов
./mvnw test

# Запуск конкретного теста
./mvnw test -Dtest=DocumentServiceIntegrationTest
```
Тесты используют Testcontainers и поднимают временную PostgreSQL.

### Покрытие тестами
- Создание документов
- Получение документов
- Пакетный SUBMIT
- Пакетный APPROVE
- Откат approve при ошибке записи в регистр

## Известные ограничения
- Пакетные операции - ограничены 1000 ID согласно ТЗ
- Реестр утверждений - сейчас в той же БД, при росте можно вынести в отдельный сервис

## Возможные улучшения при росте нагрузки
### Обработка 5000+ ID в запросе
- Использовать FORK/JOIN для параллельной обработки
- Добавить bulk insert для истории и реестра
- Увеличить batch_size в Hibernate

### Вынос реестра в отдельный сервис
- Микросервис со своей БД
- Асинхронная запись через Kafka или RabbitMQ

## Требования к окружению
- Java 17 или выше
- Docker и Docker Compose
- Maven 3.8+
